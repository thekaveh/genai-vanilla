GenAI Vanilla Stack — Architecture

Topologically-ordered · SOURCE-configurable · Kong-fronted · LiteLLM-routed

The single entry point

  • Kong gateway fronts every browser-facing service
  • *.localhost aliases routed to http://<svc>:<internal-port>
  • Basic-auth + ACLs on admin surfaces
  • Dynamic routes regenerated from service.yml on every ./start.sh

LLM Core — one API, every provider

  • LiteLLM exposes one OpenAI-compatible /v1 surface
  • Routes by model name to Ollama (local) or cloud providers
  • TEI Reranker adds a cross-encoder rerank stage for RAG
  • public.llms in Supabase is the model catalog source of truth

Agents & Workflows

  • Hermes — conversational agent runtime (Nous Research)
  • n8n — visual workflow engine, 400+ integrations
  • OpenClaw — messaging-platform AI agent
  • LightRAG — graph-augmented RAG (Neo4j + pgvector + Redis)
  • Airflow — DAG orchestration triggered via REST API

Data tier — choose your shape

  • Supabase: relational, auth, storage, realtime, studio
  • Weaviate for vectors / semantic search · pgvector fallback
  • Neo4j for knowledge graphs (Bolt + dashboard)
  • Redis for cache, sessions, websockets · MinIO for S3 blobs

Compute & Observability

  • Ray cluster — distributed Python compute (opt-in)
  • Spark cluster — master + workers + history + Connect (opt-in)
  • Prometheus scrapes Kong, LiteLLM, n8n, Backend, Weaviate, MinIO + DB sidecars
  • Grafana — pre-provisioned stack dashboards
